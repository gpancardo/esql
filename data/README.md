# ESQL

A Spanish-language dataset for translating natural language instructions into
structured outputs: **Excel formulas**, **SQL queries** (SQLite and MySQL
dialects), and **data type definitions**.

Each entry contains a user-style natural language prompt (`instruction` +
`input`) and the expected formula, query, or type definition (`output`). The
`task` field selects the target domain:

| task    | Description                           | Example output                    |
|---------|---------------------------------------|-----------------------------------|
| `excel` | Spanish Excel formulas                | `=SUMAR.SI(B:B;">5000")`         |
| `sqlite` | SQLite dialect queries                | `SELECT * FROM t WHERE x > 1;`   |
| `mysql` | MySQL dialect queries                 | `SELECT * FROM t WHERE x > 1;`      |
| `types` | Column → data type inference          | `id INTEGER\nnombre TEXT\n...`    |

## Dataset Structure

### Data Fields

- **task** (`string`): Target domain — one of `excel`, `sqlite`, `mysql`, `types`.
- **instruction** (`string`): Short description of what the user wants to accomplish.
- **input** (`string`): Natural language request with specific details (column
  names, table names, conditions, etc.).
- **output** (`string`): The expected Excel formula, SQL query, or data type
  specification.

### Data Splits

| Split | Approx. share | Stratification       |
|-------|---------------|----------------------|
| Train | 80 %          | Per-task stratified  |
| Val   | 10 %          | Per-task stratified  |
| Test  | 10 %          | Per-task stratified  |

Exact split sizes and per-task distributions are recorded in `metrics.json`.

## Usage

```python
from datasets import load_dataset

# Load all tasks (default config)
dataset = load_dataset("gpancardo/esql")

# Or load a single task
excel  = load_dataset("gpancardo/esql", "excel")
sqlite = load_dataset("gpancardo/esql", "sqlite")
mysql  = load_dataset("gpancardo/esql", "mysql")
types  = load_dataset("gpancardo/esql", "types")

# Access splits
train = dataset["train"]   # ~80 %
val   = dataset["val"]     # ~10 %
test  = dataset["test"]    # ~10 %

# Iterate
for row in train:
    print(f"[{row['task']}] {row['instruction']}")
```

Data is served in **Parquet** format for fast loading. The original JSONL files
(`train.jsonl`, `val.jsonl`, `test.jsonl`) are also available for direct use
without the `datasets` library.

## Preprocessing

The raw data (`main.jsonl`) was cleaned by:

1. Fixing malformed JSON lines.
2. Removing exact duplicates.
3. Removing entries with null or empty required fields.
4. Splitting 80-10-10 stratified by task type (seed = 42).
5. Converting JSONL splits to Parquet (full and per-task subsets).

The full cleaning + split pipeline is reproducible via:

```bash
cd data
python prepare.py
```

## Uses

### Direct Use

This dataset is intended for fine-tuning or evaluating language models on
Spanish-to-structured-output tasks:

- Translating Spanish natural language into Excel formulas.
- Generating SQL queries (SQLite / MySQL) from Spanish prompts.
- Inferring data types for database column specifications.

The dataset uses a consistent `instruction` + `input` → `output` format,
making it compatible with instruction-tuning pipelines.

### Out-of-Scope Use

- **Non-Spanish inputs:** The dataset is Spanish-only. Prompts in other
  languages will produce degraded results.
- **Production SQL:** SQL examples are syntactically valid but not
  schema-optimized. Do not use generated queries against production databases
  without review.
- **Excel localization:** Outputs use Spanish function names (`SUMAR.SI`, not
  `SUMIF`). Using this dataset for English Excel formula generation will fail.
- **Identifying individuals:** The dataset contains no real user data and
  should not be used for any deanonymization or identity-inference task.

## Dataset Creation

### Curation Rationale

This dataset was created to fill a gap in existing text-to-code resources:
there are few high-quality datasets targeting Spanish, and none that
simultaneously cover Excel formulas, SQL (two dialects), and data-type
inference in a unified format. It was designed as a fine-tuning resource for
small-to-mid-sized LMs that need to serve Spanish-speaking users.

### Source Data

The entire dataset was generated by **DeepSeek-V3** through its web chat
interface. Prompts were designed to elicit diverse, realistic examples across
all four task types. No existing datasets, web crawls, or user data were used
as source material.

#### Data Collection and Processing

1. Prompts were authored to cover a wide range of SQL patterns (joins,
   aggregations, subqueries, date functions), Excel functions (lookups,
   conditionals, math, text), and column-type inference scenarios.
2. Outputs were collected from DeepSeek-V3.
3. Raw outputs (`main.jsonl`) were cleaned by fixing malformed JSON lines,
   removing exact duplicates, and dropping entries with null or empty required
   fields.
4. The cleaned dataset was split 80-10-10 stratified by task type (seed = 42).

The full pipeline is reproducible via `python prepare.py`.

## Bias, Risks, and Limitations

### Limitations

- **Spanish only:** Outputs use Spanish function names and keywords (e.g.
  `SUMAR.SI` instead of `SUMIF`). The dataset is not suitable for
  multilingual or English-only use cases.
- **Machine-generated quality:** All examples were produced by DeepSeek-V3.
  While outputs have been spot-checked for syntactic correctness, no formal
  semantic validation has been performed. Some queries may be logically valid
  but not optimally efficient.
- **Template diversity:** As an LLM-generated dataset, some examples may
  follow repetitive structural patterns despite prompt diversity efforts.
- **Domain coverage:** The dataset covers common SQL/Excel operations but does
  not exhaustively cover edge cases, dialect-specific quirks, or every Excel
  function.

### Recommendations

- Human review is recommended for queries intended for production use.
- Users should evaluate model performance on their specific use case rather
  than relying solely on held-out metrics.
- Consider supplementing with manually curated examples for high-stakes
  applications.

### Personal and Sensitive Information

This dataset does not contain any personal, sensitive, or private information.
All data is synthetically generated and does not reference real individuals,
addresses, financial records, or any other identifiable information.
